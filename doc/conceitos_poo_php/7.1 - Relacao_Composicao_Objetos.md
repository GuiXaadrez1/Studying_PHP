## O que é composição?

Composição é uma relação estrutural forte do tipo todo-parte, onde:

- Um objeto (todo) é formado por outros objetos (partes);
- O todo controla completamente o cilco de vida das partes 
- as partes não fazem sentido existir isoladamente
- A destruição do todo implica a destruição das partes

### formalmente: 

Seja A e B dois objetos. Existe composição se:

- B é parte da estrutura de A;
- A cria, gerencia e destrói B;
- B nãa possui identidade fora de A.

## Composição NÃO é técnica, é modelo mental

Isso é crucial:

- Composição não depende de linguagem

- Não depende de private, final ou readonly

- Não depende de construtor

Ela depende exclusivamente do significado do domínio (Domian)

## Critérios científicos da composição

Para algo ser composição TODOS estes critérios devem ser verdadeiros:

| Critério                 | Obrigatório |
| ------------------------ | ----------- |
| Relação “é parte de”     | ✅           |
| Criação interna          | ✅           |
| Ciclo de vida dependente | ✅           |
| Não compartilhável       | ✅           |
| Encapsulamento           | ✅           |

Se um falhar → não é composição

## Para que a composição é usada

### Modelar estruturas reais do domínio

Composição serve para representar:

- Estruturas físicas

- Estruturas lógicas indivisíveis

- Estados internos

Exemplos conceituais:

- Pedido → Itens

- Relatório → Cabeçalho

- Usuário → Credenciais

- Conta → Extrato

- Sessão → Tokens
 
## Encapsular invariantes fortes

Composição permite garantir regras que não podem ser quebradas externamente.

Exemplo:

- Um Pedido sempre tem itens válidos

- Um CPF sempre está validado

- Um Período sempre tem início < fim

## Reduzir estados inválidos

Quanto mais composição:

- Menos setters

- Menos estados intermediários

- Menos objetos inválidos circulando

- Isso é engenharia defensiva

## Exemplo de Composição em PHP

```php
final class ItemPedido
{
    public function __construct(
        private string $produto,
        private int $quantidade
    ) {
        if ($quantidade <= 0) {
            throw new InvalidArgumentException();
        }
    }
}

final class Pedido
{
    /** @var ItemPedido[] */
    private array $itens = [];

    public function adicionarItem(string $produto, int $quantidade): void
    {
        $this->itens[] = new ItemPedido($produto, $quantidade);
    }
}
```

### Análise científica

- Pedido cria ItemPedido

- ItemPedido não é exposto para fora

- Não existe setter

- Não existe injeção

- Relação forte, fechada

👉 Composição pura

## Composição ≠ “ter um objeto como atributo”

Isto NÃO  é composição: 

```php
class Pedido
{
    public function __construct(
        private ItemPedido $item
    ) {}
}
```

Por quê?

- O item foi criado fora

- Pode ser reutilizado

- Ciclo de vida independente

👉 Associação obrigatória, não composição.

## Composição e imutabilidade  

Composição funciona melhor com obetos imutáveis:

```php
final class Periodo
{
    public function __construct(
        private DateTimeImmutable $inicio,
        private DateTimeImmutable $fim
    ) {
        if ($inicio >= $fim) {
            throw new DomainException();
        }
    }
}

final class Reserva
{
    public function __construct(
        private Periodo $periodo
    ) {}
}
```

Mesmo aqui:

Periodo é criado fora → associação

Só vira composição se Reserva criar o Periodo

## Regra de ouro definitiva

Antes de usar composição, responda:

- Isso é parte do outro?

- Pode existir sozinho?

- Pode ser compartilhado?

- Quem cria?

- Quem destrói?

Se todas as respostas apontam para dependência total → composição.
