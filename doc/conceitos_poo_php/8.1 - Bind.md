# BIND

## O que é o Bind?

Bind (vinculação) é o ato de associar uma abstração a uma implementação concreta em um contexto de resolução de depedências. 

Formalmente:

- Binda é uma regra declarativa que diz ao sistema:
- Quando alguém pedir X, forneça Y

Importante: 
- Bind não cria objetos,
- Bind não injeta objetos,
- Bind apenas declara um mapeamento

## Onde o bind existe conceitualmente

O bind existe exclusivamente em um IoC Container ou mecanismo equivalente. 

Ele não pertence: 

- À classe de domínio
- À classe de serviço 
- À regra de negócio

Ele pertence à infraestrutura. 

## O problema que o bind resolve

Sem bind: 

```php
class PedidoService
{
    public function __construct(
        // perceba que estamos fazendo a associacao/agregacao de uma class concreta
        private PedidoRepositoryMysql $repo
    ) {}
}
```
❌ Dependência concreta
❌ Viola DIP
❌ Baixa flexibilidade

Com bind

```php
class PedidoService
{
    public function __construct(
        # aplicando uma interface que com bind... acessa uma class concreta!
        # muito usado no laravel
        private PedidoRepositoryInterface $repo
    ) {}
}
```

Mas agora surge a pergunta:

    Qual implementação o sistema deve usar?

👉 Resposta: bind

## Bind na forma mais pura (Sem FrameWork)

```php
$container->bind(
    PedidoRepositoryInterface::class, # pediu a interface
    PedidoRepositoryMysql::class  # entrege a class concreta com os metodos definidos na interface

    /*
        OBSERVAÇÃO IMPORTANTE! QUANDO USAMOS ISSO:

            PedidoRepositoryMySql::class

        Estamos obtendo uma string contendo o nome completo daquela classe,
        Incluindo o Namespace. FQN - FULLY QUALIFIED NAME

        FQN -> Basicamente é o endereço completo de uma classe dentro do seu projeto

        Imageni que sua classe está em App\Repositories\PedidoRepositoryMysql.

        Ao Escrever PedidoRepositoryMysql::class, O PHP substitui isso exatamente por:
        "App\Repositories\PedidoRepositoryMysql"

        O conceito também se aplica a interfaces... Visto que estamos obtendo o FQN
    
    */
);
```

Semântica disso:

    “Sempre que alguém pedir PedidoRepositoryInterface,
    entregue PedidoRepositoryMysql.”


## Bind NÃO é injeção de dependência

| Conceito      | O que faz      |
| ------------- | -------------- |
| Bind          | Define regra   |
| DI            | Aplica a regra |
| IoC Container | Executa tudo   |

O bind não injeta nada, ele prepara o terreno

## TIPOS DE BIND

### Bind para classe concreta

```php
$container->bind(A::class,B::class)
```

- Simples
- Mais comum

### Bind para fábrica (closure)

```php
$container->bind(LoggerInterface::class,function(){
    return new FileLogger('/logs/app.log')
})
```

- Lógica Customizada
- Configuração externa
- Dependências internas

## Bind singleton

```php
$container->singleton(Config::class, function () {
    return new Config($_ENV);
});
```

- Uma instância 
- Controlado pelo container

## Bind contextual

```php
$container->when(RelatorioService::class)
          ->needs(LoggerInterface::class)
          ->give(FileLogger::class);
```

✔ Implementações diferentes conforme o contexto

## Bind e ciclo de vida

O bind não define apenas o “o quê”, mas também o “como e quando”.

| Tipo      | Ciclo                           |
| --------- | ------------------------------- |
| bind      | Nova instância a cada resolução |
| singleton | Instância única                 |
| scoped    | Uma por requisição              |

## Bind vs Factory (diferença conceitual)

| Factory           | Bind                     |
| ----------------- | ------------------------ |
| Código imperativo | Configuração declarativa |
| Local             | Global                   |
| Classe decide     | Infraestrutura decide    |

Bind substitui muitas factories manuais 

## Onde o bind deve ficar?

📌 Regra de ouro arquitetural

    Bind pertence à borda do sistema.

Em PHP:

- Bootstrap

- Service Provider

- Módulo de infraestrutura

❌ Nunca no domínio
❌ Nunca em entidades
❌ Nunca em serviços de negócio

## Exemplo completo (fluxo real)

```php

// Definicao da abastracao
interface PedidoRepositoryInterface
{
    public function salvar(Pedido $pedido): void;
}

// Implementacao concreta
class PedidoRepositoryMysql implements PedidoRepositoryInterface
{
    public function salvar(Pedido $pedido): void {}
}

// Bind no container
$container->bind(
    PedidoRepositoryInterface::class,
    PedidoRepositoryMysql::class
);

// Uso no Serviço
class PedidoService
{
    public function __construct(
        private PedidoRepositoryInterface $repo
    ) {}
}

```

📌 O serviço não sabe quem implementa
📌 O bind resolve isso externamente

## Erros comuns com bind

❌ Bind espalhado pelo sistema

    Indica arquitetura fraca.

❌ Bind em domínio

    Mistura infraestrutura com negócio.

❌ Bind sem abstração

```php
    bind(Logger::class,Logger::class);
```

    Desnecessário


## Regra mental definitiva

Bind é uma decisão de arquitetura, não de código.

## Frase Final para Internalizar

- Injeção entrega.
- Bind decide.
- Container orquestra.
